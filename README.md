做資料庫update或delete時，須注意資料表的關聯性，例如訂單明細的訂單號碼關聯於訂單主檔，不可直接刪除訂單主檔(資料庫也不允許)，需先刪除訂單明細後再刪除訂單主檔；同理update也一樣。

<p>SQL</p>
<ul>
    <li>DML：對資料本身做讀寫</li>
    <li>DDL：對資料庫物件作內容定義</li>
    <li>DCL：對資料庫做管理(權限、角色、連線)</li>
</ul>
